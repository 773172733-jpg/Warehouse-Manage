## 2. 页面迁移原则

- 不一次性移除全部mock。每个页面在对应阶段完成真实空、加载、失败和重试后再切换数据源。
- 云端响应先在service层映射为页面当前需要的view model，减少WXML大改。
- 每次写操作成功后优先使用接口返回的新余额更新当前页，再触发相关页面 `onShow` 或显式刷新。
- viewer的写入口根据可信bootstrap角色隐藏，但云函数仍独立拒绝写入。
- `NO_ACTIVE_TEAM`、`MEMBERSHIP_NOT_ACTIVE`、`TEAM_NOT_ACTIVE`和 `WAREHOUSE_NOT_FOUND`沿用现有团队上下文清理与startup重新路由。

## 3. 页面迁移顺序

### 3.1 库存首页

1. 新增 `product-service`，由 `product.list`返回items、summary和nextCursor。
2. 首屏以云端列表替换 `mock.getProducts()`；保留当前卡片view model和统一库存状态样式。
3. 搜索输入防抖后请求服务端，分类和库存状态转为查询参数，不在客户端对全量数据过滤。
4. summary使用服务端仓库级统计；产品列表分页滚动加载。
5. 明确四种状态：首屏loading、真实空仓库、筛选无结果、接口失败可重试。

### 3.2 产品详情

1. URL继续只传真实productId。
2. `product.detail`替换 `getProductById`，无效ID显示错误状态，不白屏。
3. 库存状态使用响应值，最近流水使用接口返回的最多5条摘要。
4. 产品被删除或权限失效时返回库存首页并刷新。

### 3.3 产品创建

1. 先接文字封面和emoji封面，不让图片阻塞基础创建。
2. 表单完成时生成requestKey，输入不变的重试复用。
3. 调用 `product.create`，初始库存由同一事务处理。
4. 成功后返回库存首页/上一页，通过 `onShow`刷新并定位新产品。
5. 图片模式在2C3增加prepare、上传、绑定和失败清理。

### 3.4 产品编辑与删除

1. 详情进入编辑页时携带productId，不把完整对象放URL。
2. 读取detail取得最新version，更新接口不提交stock。
3. `PRODUCT_VERSION_CONFLICT`保留用户输入，提示刷新对比，不自动覆盖。
4. 删除前显示库存要求；服务端仍以事务内stock为准。

### 3.5 库存操作

1. 页面进入时读取最新product.detail，而不是依赖上一页余额。
2. 提交前生成requestKey；弹窗取消不消耗key，业务输入改变后换key。
3. 分别调用inbound、outbound、adjust。
4. 成功后使用返回的product和record更新显示，再返回详情；超时使用同key重试。
5. `INSUFFICIENT_STOCK`、产品失效或权限变化时刷新页面状态。

### 3.6 流水页

1. `stock.record.list`替换mock数组，按createdAt游标分页。
2. 类型、产品、操作人和日期筛选改为服务端条件；关键词防抖。
3. 流水详情按recordId调用detail或使用列表完整白名单；不能从客户端修改。
4. 产品已删除时保留流水快照，隐藏“查看产品”。

## 4. 后续阶段拆分

### 阶段2C2：真实产品读与基础CRUD

范围：

- 创建 `products`和 `stock_records`集合。
- 创建2C2必需产品索引、产品流水索引、requestKey唯一索引，并关闭客户端读写权限。
- 实现 `product.create/list/detail/update/delete`，先支持文字、emoji封面。
- 接入库存首页、产品详情及基础创建/编辑/软删除。
- 初始库存大于0时必须在同一事务写入initial流水，不允许先写余额后补流水。

云端操作：新增products、stock_records集合和2C2必需索引，重新部署warehouse-api。

独立验收：owner/admin CRUD；viewer只读；搜索、分类、库存状态、统计、分页、版本冲突、软删除。

### 阶段2C3：自定义封面图片

范围：

- 实现图片prepare/upload/bind/discard与临时URL。
- 增加staging清理和旧图延迟回收策略。
- 文字和emoji流程不改。

云端操作：配置轻仓存储路径权限，新增必要的上传凭据/元数据集合或任务，重新部署warehouse-api。无需阻塞2C2。

独立验收：格式/大小限制、取消上传、创建失败孤立文件、替换图片、viewer读取、软删除保留。

### 阶段2C4：真实库存事务

范围：

- 在已创建的 `stock_records`上补齐库存查询索引。
- 实现inbound、outbound、adjust事务。
- 接入库存操作页和产品详情流水摘要。
- 完成并发出库、防超卖、事务回滚、同key重试测试。

云端操作：创建2C4新增索引并重新部署warehouse-api；集合权限沿用2C2的客户端全关闭规则。

独立验收：双账号并发、库存不足、网络重试、调整正负差、余额和流水一致。

### 阶段2C5：真实流水与搜索优化

范围：

- 接入流水列表、详情、产品/类型/操作人/日期筛选和分页。
- 创建剩余流水索引。
- 根据真实数据验证产品正则搜索P95和统计缓存。
- 只有达到阈值时才启用全文索引或独立搜索方案。

云端操作：创建2C5索引并重新部署warehouse-api；是否启用全文搜索由性能数据决定。

独立验收：历史快照、删除产品流水、组合筛选、跨页稳定性、空状态和错误恢复。

## 5. 云端人工配置时间点

当前2C1不需要创建任何集合或索引。

| 时间点 | 人工操作 | 验证 |
| --- | --- | --- |
| 2C2开发前 | 创建products、stock_records、2C2必需索引、客户端read/write=false | 索引状态可用、越权客户端访问失败 |
| 2C3联调前 | 配置轻仓云存储路径和上传限制 | 仅授权成员可上传，viewer不可写 |
| 2C4开发前 | 补齐库存操作所需索引 | 并发写入、唯一索引与事务回滚验证 |
| 2C5联调前 | 创建类型、操作人及组合筛选索引 | 查询命中且游标稳定 |

所有操作都必须在轻仓正式CloudBase环境中人工确认，云函数名称保持 `warehouse-api`。文档没有声称任何新集合或索引已创建。

## 6. 测试账号与验收方案

至少准备同一团队内三个真实微信账号：owner、admin、viewer。库存并发另用owner和admin同时操作同一产品。

### 6.1 产品验收

- owner/admin可新增、编辑、软删除零库存产品。
- viewer可搜索和查看，所有写请求返回FORBIDDEN。
- 同名产品可同时存在并通过ID分别打开。
- 最低库存边界：0库存out；正库存等于minStock为low；大于minStock为normal。
- 旧version编辑返回冲突，不覆盖其他人的更新。

### 6.2 库存验收

- 入库、出库、调整后产品余额与流水前后值严格一致。
- 双账号同时出库总量超过现存量时，最多一个/部分事务成功，最终stock不小于0。
- 人为模拟调用超时并用同requestKey重试，只产生一条流水。
- 写流水失败时余额不变化；余额更新失败时流水不创建。

### 6.3 图片验收

- 本地临时路径不进入数据库。
- JPEG/PNG/WebP范围内上传成功，超限文件拒绝。
- viewer可以看临时URL但不能上传、替换、删除。
- 创建失败的staging文件按计划回收；替换后旧图延迟删除。

### 6.4 流水验收

- 产品改名、操作人改名或离队后，旧流水保持原快照。
- 软删除产品后流水仍可查询，产品入口隐藏。
- 类型、产品、操作人、日期和关键词筛选可叠加，翻页不重复不漏项。

## 7. 已知限制与负责人确认项

以下不是2C1阻塞项，但进入对应开发阶段前需负责人确认：

1. V1每仓active产品建议上限1000是否符合首批团队规模；超过后必须提前启用搜索升级方案。
2. 产品名称和编号允许重复是否符合运营习惯；当前以ID作为唯一身份。
3. 软删除恢复是否需要在V1提供用户入口；当前只保留架构能力，不排期UI。
4. 自定义图片限制采用5MB、最长边4096像素、每产品1张是否满足需求。
5. 流水是否需要永久保留，还是未来按合规要求归档；当前设计为不修改、不删除。
6. 多仓上线时是否需要共享产品目录；若只需仓库独立产品，方案A无需迁移。
