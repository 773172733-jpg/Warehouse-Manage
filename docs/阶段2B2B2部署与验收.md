# 阶段2B2B2部署与验收

## 环境确认

1. AppID 必须为 `wxd5819a772c90b7a2`。
2. CloudBase 环境必须为 `cloud1-d8gm59cz2be4e7c23`。
3. 云函数名称必须为 `warehouse-api`。
4. 阶段2B1版本的 `warehouse-api` 已部署且三个成员接口可用。

本阶段只修改小程序端，不需要重新部署云函数，不需要新增集合、索引或权限。

## 三账号验收

准备同一团队中的 owner账号A、admin账号B和viewer账号C。

1. A进入团队页，确认真实邀请码、pending审核和active成员列表仍正常。
2. A点击自己的成员卡，确认只显示创建者信息和“暂不支持转让或退出”说明，没有管理按钮。
3. A点击C，确认显示“设为管理员”和“移出团队”。点击设为管理员并二次确认，确认提示成功，列表刷新后C显示为管理员。
4. A再次点击C，确认操作变为“取消管理员”。执行并确认列表刷新后C恢复为普通成员。
5. A点击B，选择移出团队并二次确认，确认B从真实active成员列表消失。
6. B重新打开小程序，或进入团队页并下拉刷新，确认不再显示旧团队，按startup规则进入无团队或申请状态流程。
7. C进入团队页，确认看不到邀请码、pending审核、角色修改和移除按钮，只能查看active成员。
8. C点击“退出团队”并二次确认，确认成功后自动重新bootstrap并离开旧团队页面。
9. C重新启动小程序，确认不能从库存tab继续查看旧团队内容，并按真实申请状态进入team-setup或team-join。
10. A确认底部只显示“团队创建者暂不支持退出团队”，没有可操作退出按钮。

## 异常与幂等验收

1. 角色变更或移除请求期间连续点击，确认同一目标不会重复提交。
2. 操作时断网，确认显示安全网络提示；恢复网络后对同一操作手动重试，不产生重复副作用。
3. 操作结果不确定时下拉刷新，确认页面以云端真实成员列表为准。
4. 页面和控制台不得出现 OPENID、requestKey、完整用户记录或数据库堆栈。
5. admin/viewer不能通过前端界面获得owner专属管理入口；云端仍需拒绝伪造请求。

## 回归验收

- owner生成/刷新邀请码正常。
- owner查看、批准和拒绝pending申请正常。
- 新用户申请、pending、拒绝后重申、批准后进入库存页正常。
- 创建团队与startup路由正常。
- 团队成员和操作状态不写入Storage。
- 团队页不直接调用 `wx.cloud`，也不直接访问数据库。

## 已知限制

当前不支持owner转让、解散团队、多团队切换、实时WebSocket踢出、修改成员昵称或细粒度权限。被移除用户通过重新进入、团队页刷新或受保护接口返回无权限来更新身份。
