# 阶段2C5B：低库存与缺货预警中心部署指南

## 1. 交付边界

本阶段新增当前仓库的低库存与缺货预警中心，并让库存首页的低库存、缺货统计卡进入对应筛选页面。预警是对 `warehouse_products` 当前状态的实时查询，不新增预警集合，不保存重复预警记录，不发送订阅消息。

本阶段复用 `product.list`，没有新增第二套产品列表 Action。需要重新部署 `warehouse-api` 并重新编译小程序；不需要修改 AppID、CloudBase 环境、数据库权限或云存储权限。

## 2. 预警规则

统一状态由服务端 `computeStockStatus(stock, minStock)` 计算并持久化：

- 缺货 `out`：`stock <= 0`。
- 低库存 `low`：`stock > 0` 且 `stock <= minStock`。
- 正常 `normal`：`stock > minStock`。

`minStock=0` 时：

- `stock=0` 仍为缺货。
- `stock>0` 为正常库存。
- 页面不伪造补货数量，显示“最低库存未设置，请手动评估”。

展示用建议数量为 `max(minStock - stock, 0)`，只做提示，不会自动入库。

## 3. 状态维护

产品创建、入库、出库和盘点继续复用原有统一状态计算。产品编辑现在允许修改当前仓库的 `minStock`，并在同一个产品更新事务中按当前真实 `stock` 重算 `stockStatus`。

编辑最低库存不会：

- 修改 `stock`；
- 修改 `stockVersion`；
- 新增、修改或删除 `stock_records`；
- 调用库存事务接口。

旧版客户端未提交 `minStock` 时，服务端保持原最低库存，兼容已部署调用方。

## 4. 页面入口

库存首页显示当前仓库的精确统计：

- 全部产品；
- 低库存；
- 已缺货。

点击“低库存”进入 `/pages/stock-alerts/stock-alerts?alertType=low`。

点击“已缺货”进入 `/pages/stock-alerts/stock-alerts?alertType=out`。

预警中心只有一个页面，不创建平行入口。页面支持状态切换、搜索、分类、下拉刷新、加载更多、空状态、错误重试和全仓当前状态数量。

## 5. 查询 Action

复用：

```text
product.list
```

预警页面只提交以下白名单字段：

```json
{
  "keyword": "180",
  "category": "工具",
  "stockStatus": "low",
  "cursor": "opaque-cursor",
  "pageSize": 20,
  "sort": "updated_desc",
  "includeSummary": true
}
```

说明：

- `stockStatus` 只使用 `low` 或 `out`。
- `includeSummary=true` 仅在第一页请求精确全仓统计。
- 下一页不重复请求统计。
- 客户端不能提交 `teamId`、`warehouseId`、`userId`、`openId`、`role`、`stock` 或 `stockStatus` 快照以外的库存数据。
- 团队、仓库和角色均由服务端可信用户上下文确定。

## 6. 分页和游标

- 默认 `pageSize=20`。
- 最大 `pageSize=50`。
- 排序固定为 `updatedAt DESC, _id DESC`。
- 游标包含查询版本、排序类型、最后一条 `updatedAt`、最后一条 `_id` 和查询条件哈希。
- 查询条件哈希绑定规范化关键词、搜索 token、分类、库存状态和排序。
- 切换状态、分类或搜索词会清空旧列表和旧游标。
- 加载更多只追加，并按 `warehouseProductId` 去重。

游标不使用数组下标，不会把当前页加载到前端后再做本地分页。

## 7. 搜索和分类

搜索继续复用阶段2C4A1规则及现有搜索快照：

- 忽略大小写；
- 使用 NFKC 规范化；
- 忽略常见空格、横线、下划线、斜线、点号和中点；
- 支持受控连续子串 token；
- `RA1809100` 搜索 `180` 可以命中。

搜索使用编号、名称前缀和 `searchKeywordsSnapshot` 三个索引分支。分类使用 `categorySnapshot`，可与状态和搜索组合。没有新增或修改搜索算法。

## 8. 首页统计一致性

`product.list` 在第一页按可信团队、当前仓库和 `status=active` 分别执行精确计数：

- 全部 active 仓库产品；
- `stockStatus=low`；
- `stockStatus=out`。

统计与预警列表使用同一团队、仓库、active 状态和库存状态语义，不再使用当前已加载页数量冒充全仓统计。

合法业务流程保证共享目录删除前所有仓库实例都已移出。预警列表还会批量核对 `products.status=active`，不展示不可访问的产品。

## 9. 权限

- owner：可查看、进入详情，并显示“入库”按钮。
- admin：可查看、进入详情，并显示“入库”按钮。
- viewer：可查看和进入允许查看的详情，不显示“入库”按钮。
- pending/removed：服务端返回 `FORBIDDEN`，不能查看预警中心。

即使 viewer 直接输入库存操作页路径，现有库存 Action 仍要求 admin 级权限并返回 `FORBIDDEN`。

“入库”按钮进入现有库存操作页。该页会调用 `product.detail` 重新读取权威库存、权限和 `stockVersion`，不会使用预警列表快照直接提交。

## 10. 数据隔离和封面

服务端只查询：

- 可信用户当前团队；
- 当前有效仓库；
- `status=active` 的 `warehouse_products`；
- 仍为 active 且允许展示的 `products`。

当前页产品资料通过一次 `command.in(productIds)` 批量读取，不存在逐商品 N+1 查询。

封面继续支持系统贴图、文字、默认封面和自定义图片。私有图片由现有批量安全逻辑换取临时 HTTPS URL；响应不返回完整 `fileID`，不拼接公开 URL，不修改图片清理 Worker。

## 11. 使用的索引

本阶段不新增、删除、重排或自动创建索引。继续使用 `database/indexes.md` 中已经人工创建的 `warehouse_products` 索引。

基础列表和精确统计：

```text
idx_wh_products_status_updated
teamId ASC
warehouseId ASC
status ASC
updatedAt DESC
_id DESC
非唯一
```

状态筛选和状态统计：

```text
idx_wh_products_stock_status
teamId ASC
warehouseId ASC
status ASC
stockStatus ASC
updatedAt DESC
_id DESC
非唯一
```

状态+分类：

```text
idx_wh_products_category_stock
teamId ASC
warehouseId ASC
status ASC
categorySnapshot ASC
stockStatus ASC
updatedAt DESC
_id DESC
非唯一
```

搜索组合继续使用：

- `idx_wh_products_stock_name`；
- `idx_wh_products_stock_code`；
- `idx_wh_products_stock_keyword`；
- `idx_wh_products_category_stock_name`；
- `idx_wh_products_category_stock_code`；
- `idx_wh_products_category_stock_keyword`。

这些索引都按等值字段在前、`updatedAt DESC` 和 `_id DESC` 在后，均为非唯一。正式环境无需用户再创建索引。

## 12. 刷新机制

以下成功操作会设置内存刷新标记：

- 产品创建；
- 产品编辑和最低库存修改；
- 入库；
- 出库；
- 盘点；
- 产品移出当前仓库；
- 产品恢复到当前仓库。

预警页 `onShow` 消费标记并重新加载第一页。标记只在当前小程序进程内使用，不写 Storage。页面卸载后通过活动状态和请求版本阻止 `setData`。

## 13. 错误处理

页面保留业务错误差异：

- `FORBIDDEN`：无权查看或操作；
- `WAREHOUSE_NOT_ACTIVE`：当前仓库不可用；
- `INVALID_CURSOR`：筛选或分页状态失效，重新加载第一页；
- `INVALID_CATEGORY`：分类无效；
- `PRODUCT_NOT_FOUND` / `PRODUCT_NOT_ACTIVE`：产品不可用；
- `CLOUD_CALL_FAILED`：网络连接失败；
- `INTERNAL_ERROR` / `DATABASE_ERROR`：服务端处理失败。

预警页参数只接受 `low` 和 `out`；无效页面参数安全回退到 `low`。服务端直接复用 `product.list`，非法 `stockStatus` 返回专用 `INVALID_ALERT_TYPE`，不会被笼统转换为 `INVALID_INPUT`。

## 16. 手动验收

1. 使用 owner 登录，在产品编辑页把一个有库存商品的最低库存调到等于或高于当前库存。
2. 返回库存首页，确认低库存精确数量增加。
3. 点击首页“低库存”，确认进入预警中心且该商品出现。
4. 把最低库存调低到小于当前库存，确认商品离开低库存列表且首页数量同步减少。
5. 将一个商品盘点为0，确认只进入“已缺货”，不进入“低库存”。
6. 验证 `minStock=0`、`stock=0` 仍显示缺货，并提示手动评估补货量。
7. 对名称或型号为 `RA1809100` 的预警商品搜索 `180`，确认命中。
8. 分别验证 `RA 180`、不同大小写和常见分隔符。
9. 组合验证状态、分类和搜索。
10. 创建超过20条同状态测试商品，验证加载下一页不重复、不遗漏。
11. owner/admin 点击“入库”，确认进入现有入库页；入库后返回，预警列表和首页统计刷新。
12. viewer 登录，确认可以查看预警和详情，但页面不显示“入库”按钮。
13. viewer 直接输入库存操作页路径，确认服务端拒绝。
14. pending/removed 成员尝试进入，确认被拒绝。
15. 验证文字封面、系统贴图、自定义图片和图片失败降级。
16. 在云函数返回和日志中确认没有 `teamId`、`warehouseId`、用户身份、请求标识、`fileID` 或临时图片 URL 泄漏。

验收后删除仅为测试创建的数据时，必须使用正式业务流程；不要直接删除或修改历史库存流水。

不要回滚、编辑或删除 `stock_records`，不要修改库存余额，不要执行破坏性数据库操作。

## 18. 已知限制

- 当前只提供页面内实时预警，不发送订阅消息、短信或邮件。
- 推荐补货量只基于最低库存线，不包含采购周期、销量预测或安全库存模型。
- “全仓当前状态”是状态维度的精确总数；叠加搜索或分类后不冒充组合筛选总数，列表仍通过分页展示全部匹配项。
- 历史产品若尚未执行阶段2C4A1搜索字段回填，连续型号片段可能无法命中，应使用已经审计的 `product.search.rebuild`，不得前端全量扫描。
- 合法业务写入维护 `products` 与 `warehouse_products` 状态不变量；禁止控制台人工改写业务数据。
