# 阶段2C3A部署与验收

## 部署前确认

1. 确认轻仓正式AppID、独立CloudBase环境和云函数名称均与项目正式配置一致。
2. 确认products、warehouse_products、stock_records集合仍为客户端read=false、write=false。
3. 阶段2C3A不创建新集合、不修改数据库权限、不新增索引。回收站复用 `idx_wh_products_status_updated` 及现有分类/搜索组合索引。

## 必须部署

在微信开发者工具中右键 `cloudfunctions/warehouse-api`，选择“上传并部署：云端安装依赖”。等待部署成功后再编译小程序。不要创建第二个云函数，也不要覆盖其他小程序环境。

## 编辑验收

1. owner或admin打开零库存或有库存产品详情，选择“编辑产品”。
2. 修改名称、编号、分类、单位、品牌、规格、介绍或文字/emoji/none封面。
3. 确认库存步骤只读并显示“库存和预警值需要通过库存操作功能修改”。
4. 保存后详情重新读取最新资料，库存首页重新显示最新快照。
5. 在云控制台确认products.version只加1，normalizedName、normalizedCode、searchKeywords由服务端生成。
6. 确认warehouse_products快照和productVersion同步，但stock、minStock、stockVersion不变，stock_records没有新增。

制造version冲突：账号A和账号B同时打开同一产品编辑页；A先保存，B再保存。B必须看到“产品已被其他成员修改，请刷新后重新编辑”，且不能覆盖A的结果。选择重新加载后再修改保存。

## 移除与恢复验收

1. 对stock大于0的产品选择“从当前仓库移除”，页面应提前阻断；云端直接调用也应返回PRODUCT_HAS_STOCK。
2. 选择stock=0的产品，确认提示明确保留历史流水，然后执行移除。
3. 库存首页不再显示该产品；products文档仍active，activeWarehouseCount减1。
4. 原warehouse_products文档ID不变、status=removed、stock=0，并有removedAt/removalReason；teams.activeProductCount不变。
5. 记录移除前stock_records数量和ID，确认移除后没有删除或新增。
6. 从库存首页进入“回收站”，检查搜索、分类、下拉刷新和空/错误状态。
7. 点击恢复并确认。恢复后原warehouseProductId不变、status=active、stock=0、stockStatus=out，快照来自最新products，activeWarehouseCount加1。
8. 再次确认stock_records数量和ID完全不变，返回库存页后产品重新出现。

## 权限验收

- owner/admin：可编辑、零库存移除、查看回收站和恢复。
- viewer：详情没有编辑/移除入口，库存页没有回收站入口；手动编译回收站或编辑路径也会被页面和云端拒绝。
- pending/removed成员：不能读取或写入团队产品数据。

## 当前不包含

全局共享目录删除/恢复属于2C3B；图片上传/替换属于2C3C；入库、出库、库存调整和真实流水列表仍未接入。
