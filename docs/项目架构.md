# 项目架构

## 小程序端

`miniprogram/app.js` 负责初始化 CloudBase，并调用 `userService.bootstrap()` 建立本地启动状态。

`miniprogram/pages/startup` 是启动中转页，不放入 tabBar。初始化成功后跳转到库存页，失败时展示 `error-state` 并支持重新加载。

四个 tab 页面为：

- `pages/inventory/inventory`
- `pages/records/records`
- `pages/team/team`
- `pages/profile/profile`

阶段0页面只做占位，不实现正式业务。

## 前端基础层

- `services/cloud-service.js`：封装 `wx.cloud.callFunction`，统一调用 `api` 云函数。
- `services/user-service.js`：返回本地占位用户状态。
- `constants/`：角色、路由和错误码。
- `utils/permission.js`：仅用于页面展示控制，真实权限必须由云函数验证。
- `utils/error-handler.js`：将异常转换为可展示状态。

## 云函数

`cloudfunctions/api` 是统一云函数入口。请求格式：

```json
{
  "action": "system.ping",
  "data": {}
}
```

响应格式固定为成功或失败两类，均包含 `requestId` 和 `timestamp`。

`router.js` 使用明确 action 白名单，不允许动态 require 任意路径。阶段0仅开放 `system.ping`。
