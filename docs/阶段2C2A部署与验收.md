# 阶段2C2A部署与验收

## 1. 操作环境

只操作轻仓：

- AppID：`wxd5819a772c90b7a2`
- CloudBase：`cloud1-d8gm59cz2be4e7c23`
- 云函数：`warehouse-api`

不得操作事件管理器项目（AppID 首段 `wx6628****`、CloudBase 首段 `cloud1-d4g6****`）。本文记录的是待执行步骤，不代表云端资源已经创建或部署。

## 2. 创建集合

在轻仓云开发控制台的数据库中创建：

1. `products`
2. `warehouse_products`
3. `stock_records`

不要导入阶段1 mock数据，不要手工添加业务产品。

## 3. 创建索引

严格按 `database/indexes.md` 创建标为“2C2A部署前必须”的索引：

- products：6个。
- warehouse_products：唯一关系、创建幂等、基础列表、分类/库存组合，以及名称、编号、关键词与可选筛选组合索引。
- stock_records：`idx_records_wh_created`、`idx_records_wh_product_created`、`idx_records_product_created`、`uidx_records_request`。

字段顺序和升降序必须完全一致，只给 `uidx_` 索引开启唯一。等待每个索引状态变为“可用”后再继续。若控制台对OR查询提示缺少索引，先核对查询条件与文档，不能改成客户端全量加载规避。

## 4. 设置权限

三个新集合分别设置自定义安全规则：

```json
{
  "read": false,
  "write": false
}
```

保存后确认小程序客户端不能直接读写；该规则不影响warehouse-api使用服务端SDK。

## 5. 部署云函数

1. 在微信开发者工具确认当前项目和云环境均为轻仓。
2. 右键 `cloudfunctions/warehouse-api`。
3. 选择“上传并部署：云端安装依赖”。
4. 等待部署成功，不要选择事件管理器的云函数或环境。
5. 本阶段不需要创建新的云函数名称。

## 6. owner/admin创建验证

使用已在真实团队中的owner账号，在开发者工具调试器执行一次测试调用：

```js
wx.cloud.callFunction({
  name: 'warehouse-api',
  data: {
    action: 'product.create',
    data: {
      name: '2C2A测试扳手',
      productCode: 'TEST-2C2A-001',
      category: '工具',
      unit: '把',
      coverType: 'text',
      coverText: '扳手',
      coverBackground: '#EAF6EF',
      minStock: 5,
      initialStock: 8,
      requestKey: 'product_2c2a_test_001'
    }
  }
}).then(console.log)
```

确认响应success为true、idempotent为false，并记录返回的productId和warehouseProductId。

## 7. 数据一致性验证

在控制台只读检查：

1. products恰好新增1条，且无warehouseId、stock、minStock、stockStatus、stockVersion。
2. warehouse_products恰好新增1条，teamId/warehouseId来自真实上下文，stock=8、minStock=5、stockStatus=normal。
3. stock_records恰好新增1条initial流水，beforeStock=0、afterStock=8、changeQuantity=8。
4. teams.activeProductCount比测试前增加1。
5. 三条业务记录的产品和仓库实例ID相互一致。

再创建initialStock=0的第二个测试产品，确认没有initial流水但products、warehouse_products和团队计数正常提交。

## 8. 幂等验证

1. 原样重发第一条product.create，确认success=true、idempotent=true，三个集合均不新增重复记录，团队计数不再增加。
2. 保持requestKey不变，仅修改initialStock或名称，确认返回 `REQUEST_KEY_CONFLICT`。
3. 不要把requestKey或完整产品对象复制到公开日志或截图。

## 9. 列表与详情验证

分别调用：

```js
wx.cloud.callFunction({
  name: 'warehouse-api',
  data: { action: 'product.list', data: { pageSize: 1 } }
}).then(console.log)
```

把返回的nextCursor原样传入下一次请求，确认无重复或漏项。再测试编号精确搜索、名称前缀、category和stockStatus。使用返回的productId调用product.detail，确认只返回当前仓库库存且permissions符合角色。

## 10. viewer权限验证

使用同团队viewer账号：

1. product.list成功。
2. product.detail成功，三个写权限标记均为false。
3. product.create返回 `FORBIDDEN`，数据库和团队计数不变化。

pending、removed成员调用列表和详情应被拒绝。

## 11. 测试数据清理

阶段2C2A尚无删除接口。只允许在专用测试团队中通过云控制台清理：

1. 记录测试前teams.activeProductCount。
2. 按 `stock_records -> warehouse_products -> products` 顺序删除本次明确记录的测试ID。
3. 将测试团队activeProductCount恢复到测试前值并更新updatedAt。
4. 再次确认没有删除非测试数据。

正式业务团队的流水按永久保留规则不得这样清理；需要保留测试数据或等待后续正式软删除接口。
