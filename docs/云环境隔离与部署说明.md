# 云环境隔离与部署说明

## 强制隔离规则

1. 事件管理器与仓库管理器禁止共用生产 CloudBase 环境。
2. 仓库管理器必须使用独立的小程序 AppID 和独立 CloudBase 环境。
3. 仓库管理器统一云函数名称固定为 `warehouse-api`。
4. 禁止将 `warehouse-api` 部署到事件管理器环境。
5. 没有独立环境前，只允许运行本地 UI 骨架，不得调用云函数或创建集合。
6. 禁止在事件管理器环境创建任何仓库相关集合。

## 部署前核对

每次部署必须逐项核对并记录：

- 小程序 AppID 属于仓库管理器。
- 云环境名称属于仓库管理器。
- 云环境 ID 与 `WAREHOUSE_CLOUD_ENV` 完全一致。
- 待部署云函数名称为 `warehouse-api`。

任一项无法确认时立即停止部署。不得使用事件管理器环境进行临时测试。

## 首次配置与部署

1. 由项目所有者在微信公众平台准备仓库管理器专用 AppID。
2. 在微信开发者工具中使用该 AppID 导入本项目。
3. 由项目所有者创建仓库管理器专用 CloudBase 环境。
4. 将环境 ID 填入 `miniprogram/config/env.js` 的 `WAREHOUSE_CLOUD_ENV`。
5. 确认开发者工具当前环境与配置一致。
6. 右键 `cloudfunctions/warehouse-api`，选择“上传并部署：云端安装依赖”。

部署不会自动创建数据库集合。数据库、索引和权限规则必须在后续获批阶段独立执行。
