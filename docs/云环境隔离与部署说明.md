# 云环境隔离与部署说明

## 轻仓正式环境

- 小程序 AppID：`wxd5819a772c90b7a2`
- CloudBase 环境名称：`cloud1`
- CloudBase 环境 ID：`cloud1-d8gm59cz2be4e7c23`
- 统一云函数名称：`warehouse-api`

以上配置只属于轻仓。部署、创建集合或修改权限前，必须在微信开发者工具和云开发控制台同时核对 AppID 与环境 ID。

## 强制隔离规则

1. 事件管理器与仓库管理器禁止共用生产 CloudBase 环境。
2. 仓库管理器必须使用独立的小程序 AppID 和独立 CloudBase 环境。
3. 仓库管理器统一云函数名称固定为 `warehouse-api`。
4. 禁止将 `warehouse-api` 部署到事件管理器环境。
5. 配置缺失或环境不匹配时，不得调用云函数或创建集合。
6. 禁止在事件管理器环境创建任何仓库相关集合。

## 部署前核对

每次部署必须逐项核对并记录：

- 小程序 AppID 属于仓库管理器。
- 云环境名称属于仓库管理器。
- 云环境 ID 与 `WAREHOUSE_CLOUD_ENV` 完全一致。
- 待部署云函数名称为 `warehouse-api`。

任一项无法确认时立即停止部署。不得使用事件管理器环境进行临时测试。

## 首次配置与部署

1. 在微信开发者工具中确认项目 AppID 为 `wxd5819a772c90b7a2`。
2. 打开云开发控制台，确认当前环境 ID 为 `cloud1-d8gm59cz2be4e7c23`。
3. 确认 `miniprogram/config/env.js` 的 `WAREHOUSE_CLOUD_ENV` 与控制台完全一致。
4. 确认开发者工具当前环境与配置一致。
5. 右键 `cloudfunctions/warehouse-api`，选择“上传并部署：云端安装依赖”。

部署不会自动创建数据库集合。阶段2A的集合、索引和权限规则必须按 [`阶段2A部署与验收.md`](阶段2A部署与验收.md) 人工配置并核对。

## 本机配置

`project.private.config.json` 已被 `.gitignore` 忽略，用于保存微信开发者工具生成的本机编译偏好。正式 AppID 保存在 `project.config.json`；Secret、密钥、密码和腾讯云访问凭证不得写入任何项目配置文件。
