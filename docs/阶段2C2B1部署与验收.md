# 阶段2C2B1部署与验收

## 部署说明

本阶段仅修改小程序端产品创建页面、库存页新增入口权限和本地文档/测试。若阶段2C2A的 `warehouse-api`、三个集合和必需索引已经部署，本阶段无需重新部署云函数，无需新增集合、索引或权限。

开始验收前确认：

- AppID为 `wxd5819a772c90b7a2`。
- CloudBase环境为轻仓独立环境。
- `warehouse-api` 已包含阶段2C2A的 `product.create`。
- `products`、`warehouse_products`、`stock_records` 已创建且客户端直访关闭。

## owner/admin创建测试

1. 使用owner账号进入库存页，确认右下角显示新增按钮。
2. 依次填写基本信息、库存设置并进入确认预览。
3. 使用文字封面创建一个初始库存为 `0` 的产品。
4. 确认出现“产品创建成功”并返回库存Tab。
5. 在云开发控制台确认 `products` 和 `warehouse_products` 各新增一条，`stock_records` 不新增initial流水。
6. 再使用系统emoji封面创建初始库存大于 `0` 的产品。
7. 确认三个集合均新增对应记录，initial流水的变更前库存为0、变更后库存等于初始库存。
8. 使用admin账号重复一次创建，确认允许操作。

产品名称和编号允许重复，不应使用名称是否重复作为验收失败依据。

## viewer权限测试

1. 使用viewer账号进入库存页，确认不显示新增产品按钮。
2. 通过开发者工具直接编译 `pages/product-edit/product-edit?mode=create`。
3. 确认页面侧提示无权限并返回库存页。
4. 后端仍应拒绝任何绕过页面发起的viewer创建请求。

## 幂等重试测试

1. 填好表单并在确认页点击完成。
2. 在请求期间快速连续点击，确认只发起一次创建。
3. 使用开发者工具网络限制制造一次失败，再恢复网络并直接重试。
4. 确认相同表单只生成一个产品、一个仓库关系，以及最多一条initial流水。
5. 网络失败后修改任一普通字段再提交，确认能够使用新请求意图创建。
6. 若出现 `REQUEST_KEY_CONFLICT`，再次点击应使用新键提交，不会无限自动重试。

页面不会显示、记录或持久化 `requestKey`。

## 本地图片阻断测试

1. 在封面步骤选择自定义图片并保留本地预览。
2. 到确认页点击完成。
3. 确认显示“自定义图片上传将在后续阶段开放，请先使用文字或表情封面。”
4. 确认没有调用云端创建，表单内容仍在，可返回修改封面。

## 错误与边界测试

- 初始库存和最低库存输入小数、负数、非数字或超大整数时不能提交。
- 产品数量达到上限时提示“团队产品数量已达到上限”。
- 云端不可用时保留表单并提示检查网络。
- 当前团队或仓库失效时，应用刷新bootstrap并重新进入startup流程。
- 创建成功后只能使用 `switchTab` 返回库存页，不应再出现旧占位提示。

## 当前限制

库存首页仍为mock数据，创建成功后不会立即在列表出现。不要向mock列表写入云端返回对象，也不要用Storage保存产品。阶段2C2B2将接入真实 `product.list` 和 `product.detail`。
