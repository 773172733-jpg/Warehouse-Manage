# 阶段2C1接口契约

## 1. 通用约定

所有接口继续通过统一云函数 `warehouse-api` 调用：

```js
wx.cloud.callFunction({
  name: 'warehouse-api',
  data: {
    action: 'product.list',
    data: {},
    requestId: 'client_request_id'
  }
});
```

成功响应：

```json
{
  "success": true,
  "requestId": "req_xxx",
  "timestamp": 1784217600000,
  "data": {}
}
```

失败响应：

```json
{
  "success": false,
  "requestId": "req_xxx",
  "timestamp": 1784217600000,
  "error": {
    "code": "PRODUCT_NOT_FOUND",
    "message": "产品不存在或已不可用。"
  }
}
```

### 1.1 可信上下文

每个handler都必须从 `cloud.getWXContext()` 获取可信OPENID，再查询active用户、active成员关系、active团队和active仓库。客户端不得传入并被信任：

- `teamId`
- `warehouseId`
- `userId`
- `operatorId`
- `role`
- `openId`
- `createdBy`
- `updatedBy`

发现这些字段时直接返回 `FORBIDDEN`，而不是静默忽略。`productId`、`recordId`、`memberId`仅是候选资源定位字段，服务端仍须验证资源属于可信团队和仓库。

### 1.2 权限等级

- owner：admin及以下全部业务能力，并保留团队管理能力。
- admin：产品写、库存写、产品和流水读。
- viewer：产品、库存和流水只读。
- pending、removed或disabled用户：无团队业务读取和写入权限。

### 1.3 分页

- 默认 `pageSize=20`，最大50。
- 产品游标按 `updatedAt desc, _id desc`；流水游标按 `createdAt desc, _id desc`。
- `nextCursor`是服务端编码值，客户端原样回传，不解析、不修改。
- 无更多数据时 `nextCursor=null`；空列表返回 `items: []`，不是错误。

### 1.4 幂等

- 所有写接口要求 `requestKey`，格式与现有项目一致：8至64字符，只允许字母、数字、下划线和连字符。
- 网络超时重试复用原key；用户修改业务输入后生成新key。
- 服务端对规范化业务输入计算哈希。同key同输入返回首次结果；同key不同输入返回 `DUPLICATE_REQUEST`。
- 读接口不要求requestKey。

## 2. 产品接口

### 2.1 product.create

权限：owner、admin。

请求：

```json
{
  "name": "800×800浅灰柔光砖",
  "code": "GZ-800-H",
  "category": "瓷砖",
  "unit": "箱",
  "brand": "冠珠",
  "specification": "800×800mm",
  "description": "浅灰色柔光砖",
  "keywords": ["柔光", "浅灰", "地砖"],
  "cover": {
    "type": "text",
    "text": "800×800",
    "emoji": "",
    "assetKey": "",
    "fileId": "",
    "background": "#EAF6EF",
    "uploadToken": ""
  },
  "initialStock": 126,
  "minStock": 30,
  "requestKey": "product_create_xxx"
}
```

规则：

- `name`、`category`、`unit`必填；名称允许重复。
- `initialStock`、`minStock`为0至999999999的整数。
- `cover.type=image`时必须先完成受控上传，并提交有效fileID和uploadToken。
- `initialStock>0`时，产品与 `initial` 流水在同一事务创建；等于0时不建流水。
- 客户端不能提交 `stockStatus`、`version`、`status`或身份字段。

返回：

```json
{
  "product": {
    "id": "prod_xxx",
    "name": "800×800浅灰柔光砖",
    "code": "GZ-800-H",
    "category": "瓷砖",
    "unit": "箱",
    "stock": 126,
    "minStock": 30,
    "stockStatus": "normal",
    "version": 1,
    "stockVersion": 1,
    "cover": {
      "type": "text",
      "text": "800×800",
      "emoji": "",
      "background": "#EAF6EF",
      "url": ""
    },
    "createdAt": "server-date",
    "updatedAt": "server-date"
  },
  "initialRecord": {
    "id": "record_xxx",
    "type": "initial",
    "changeQuantity": 126,
    "beforeStock": 0,
    "afterStock": 126
  }
}
```

主要错误：`INVALID_PRODUCT_NAME`、`INVALID_CATEGORY`、`INVALID_UNIT`、`INVALID_DESCRIPTION`、`INVALID_COVER`、`INVALID_STOCK_QUANTITY`、`INVALID_MIN_STOCK`、`DUPLICATE_REQUEST`、`FORBIDDEN`。

### 2.2 product.list

权限：owner、admin、viewer。

请求：

```json
{
  "keyword": "柔光",
  "category": "瓷砖",
  "stockStatus": "low",
  "sort": "updated_desc",
  "pageSize": 20,
  "cursor": "opaque_cursor"
}
```

规则：

- `keyword`最多40字符，搜索规范化名称、编号和关键词，不搜索介绍。
- `stockStatus`只允许空、`out`、`low`、`normal`。
- V1排序支持 `updated_desc`；`stock_asc`、`stock_desc`待专用索引创建后开放。
- `status`固定为active，客户端不能查询回收站。
- summary是当前可信仓库全部active产品统计，不随keyword和分类变化，匹配当前首页语义。

返回：

```json
{
  "items": [
    {
      "id": "prod_xxx",
      "name": "美纹纸胶带",
      "code": "MT-025",
      "category": "耗材",
      "unit": "卷",
      "stock": 8,
      "minStock": 15,
      "stockStatus": "low",
      "cover": {
        "type": "emoji",
        "text": "",
        "emoji": "📦",
        "background": "#EAF6EF",
        "url": ""
      },
      "updatedAt": "server-date"
    }
  ],
  "summary": {
    "total": 12,
    "low": 3,
    "out": 2
  },
  "nextCursor": null
}
```

空状态：`items: []`；若仓库没有任何active产品，summary全部为0。

主要错误：`INVALID_INPUT`、`NO_ACTIVE_TEAM`、`WAREHOUSE_NOT_FOUND`、`FORBIDDEN`、`DATABASE_ERROR`。

### 2.3 product.detail

权限：owner、admin、viewer。

请求：

```json
{
  "productId": "prod_xxx",
  "includeRecentRecords": true
}
```

返回完整展示字段、库存字段、创建/更新时间和最多5条最近流水摘要。owner/admin额外获得用于编辑的 `version`；viewer不需要内部版本和任何审计ID。

```json
{
  "product": {
    "id": "prod_xxx",
    "name": "美纹纸胶带",
    "code": "MT-025",
    "category": "耗材",
    "unit": "卷",
    "brand": "3M",
    "specification": "25mm×50m",
    "description": "耐高温、不留残胶",
    "keywords": ["胶带", "遮蔽"],
    "stock": 8,
    "minStock": 15,
    "stockStatus": "low",
    "version": 2,
    "stockVersion": 4,
    "cover": {
      "type": "text",
      "text": "MT",
      "emoji": "",
      "background": "#EAF6EF",
      "url": ""
    },
    "createdAt": "server-date",
    "updatedAt": "server-date"
  },
  "recentRecords": []
}
```

deleted产品返回 `PRODUCT_NOT_ACTIVE`，不把完整删除文档返回前端。

主要错误：`PRODUCT_NOT_FOUND`、`PRODUCT_NOT_ACTIVE`、`FORBIDDEN`、`DATABASE_ERROR`。

### 2.4 product.update

权限：owner、admin。

请求：

```json
{
  "productId": "prod_xxx",
  "version": 2,
  "name": "美纹纸胶带 25mm",
  "code": "MT-025",
  "category": "耗材",
  "unit": "卷",
  "brand": "3M",
  "specification": "25mm×50m",
  "description": "更新后的介绍",
  "keywords": ["胶带", "遮蔽"],
  "minStock": 12,
  "cover": {
    "type": "text",
    "text": "MT",
    "emoji": "",
    "assetKey": "",
    "fileId": "",
    "background": "#EAF6EF",
    "uploadToken": ""
  },
  "requestKey": "product_update_xxx"
}
```

规则：

- 更新采用完整可编辑字段提交，服务端白名单解析。
- 请求中禁止出现 `stock`、`stockStatus`、`stockVersion`；库存只能通过inventory接口变化。
- `minStock`可以修改，但服务端必须在事务中按当前stock重算状态并递增stockVersion。
- `version`不匹配时返回 `PRODUCT_VERSION_CONFLICT`，不得自动覆盖。

返回更新后的产品白名单。网络超时后使用同一requestKey重试；若已经发生后续修改，则读取detail确认当前版本。

### 2.5 product.delete

权限：owner、admin。

请求：

```json
{
  "productId": "prod_xxx",
  "version": 3,
  "requestKey": "product_delete_xxx"
}
```

规则：

- 只做软删除。
- `stock>0`返回 `PRODUCT_HAS_STOCK`。
- 删除不修改、不删除历史流水；不立即删除自定义封面。
- 同key重试返回相同删除结果。

返回：

```json
{
  "product": {
    "id": "prod_xxx",
    "status": "deleted",
    "deletedAt": "server-date"
  }
}
```

## 3. 库存接口

三种库存接口都要求owner/admin权限，必须在同一数据库事务中更新产品余额并创建流水。viewer调用统一返回 `FORBIDDEN`。

### 3.1 inventory.inbound

请求：

```json
{
  "productId": "prod_xxx",
  "quantity": 20,
  "sourceOrDestination": "佛山供应商",
  "remark": "七月补货",
  "requestKey": "inventory_inbound_xxx"
}
```

`quantity`必须为正整数。`reason`由服务端固定为“入库”或根据受限业务枚举生成，不允许客户端伪造操作类型。

### 3.2 inventory.outbound

请求：

```json
{
  "productId": "prod_xxx",
  "quantity": 3,
  "sourceOrDestination": "滨江项目部",
  "remark": "施工领用",
  "requestKey": "inventory_outbound_xxx"
}
```

锁定产品后再次校验库存。并发情况下以事务内最新stock为准；不足返回 `INSUFFICIENT_STOCK`，余额与流水都不写入。

### 3.3 inventory.adjust

请求：

```json
{
  "productId": "prod_xxx",
  "targetStock": 18,
  "reason": "盘点修正",
  "remark": "补录未登记到货",
  "requestKey": "inventory_adjust_xxx"
}
```

`targetStock`必须为非负整数，`reason`必填且最多40字符。目标等于当前库存时返回 `INVALID_STOCK_QUANTITY`，不创建零变化流水。

### 3.4 库存接口统一返回

```json
{
  "product": {
    "id": "prod_xxx",
    "stock": 18,
    "minStock": 15,
    "stockStatus": "normal",
    "stockVersion": 5,
    "updatedAt": "server-date"
  },
  "record": {
    "id": "record_xxx",
    "type": "adjust",
    "changeQuantity": 10,
    "beforeStock": 8,
    "afterStock": 18,
    "reason": "盘点修正",
    "remark": "补录未登记到货",
    "operator": {
      "memberId": "member_xxx",
      "displayName": "微信用户"
    },
    "createdAt": "server-date"
  }
}
```

主要错误：`PRODUCT_NOT_FOUND`、`PRODUCT_NOT_ACTIVE`、`INVALID_STOCK_QUANTITY`、`INSUFFICIENT_STOCK`、`DUPLICATE_REQUEST`、`FORBIDDEN`、`WAREHOUSE_NOT_FOUND`、`DATABASE_ERROR`。

## 4. 流水接口

### 4.1 stock.record.list

权限：owner、admin、viewer。

请求：

```json
{
  "keyword": "滨江",
  "productId": "prod_xxx",
  "type": "outbound",
  "operatorMemberId": "member_xxx",
  "dateFrom": "2026-07-01T00:00:00.000Z",
  "dateTo": "2026-08-01T00:00:00.000Z",
  "pageSize": 20,
  "cursor": "opaque_cursor"
}
```

规则：

- `type`允许 `initial`、`inbound`、`outbound`、`adjust`。
- `operatorMemberId`只是筛选候选，服务端先验证成员记录属于当前团队，再映射内部operatorId。
- 客户端不能传 `operatorId`。
- keyword最多40字符，搜索产品名/编号快照、操作人名快照、原因、来源去向和备注。V1仅在受控数据规模内使用。
- 日期采用左闭右开区间；最长查询范围建议不超过366天。

返回：

```json
{
  "items": [
    {
      "id": "record_xxx",
      "product": {
        "id": "prod_xxx",
        "name": "美纹纸胶带",
        "code": "MT-025"
      },
      "type": "outbound",
      "changeQuantity": -3,
      "beforeStock": 11,
      "afterStock": 8,
      "unit": "卷",
      "reason": "出库",
      "sourceOrDestination": "滨江项目部",
      "remark": "施工领用",
      "operator": {
        "memberId": "member_xxx",
        "displayName": "微信用户"
      },
      "createdAt": "server-date",
      "productActive": true
    }
  ],
  "nextCursor": null
}
```

空状态返回 `items: []`。流水不可编辑和删除。

### 4.2 stock.record.detail

权限：owner、admin、viewer。

请求：

```json
{
  "recordId": "record_xxx"
}
```

服务端按 `_id`读取后验证 `teamId`和 `warehouseId`属于可信上下文，返回与列表相同的完整白名单及 `productActive`。不返回内部operatorId、requestKey、输入哈希或数据库原始文档。

主要错误：`STOCK_RECORD_NOT_FOUND`、`FORBIDDEN`、`DATABASE_ERROR`。

## 5. 封面辅助接口

图片不阻塞文字和emoji产品创建。阶段2C3可增加：

### 5.1 product.cover.prepare

权限：owner、admin。请求只包含文件扩展名、MIME、大小和requestKey。云端返回短期uploadToken与轻仓环境内的staging cloudPath。客户端上传后，`product.create/update`验证fileID与token绑定关系。

### 5.2 product.cover.discard

权限：原上传者owner/admin。用于主动清理未绑定staging文件；即使不调用，也由24小时清理任务回收。

两个接口都不能在数据库事务中上传或删除文件。viewer只通过产品读接口获得临时URL。

## 6. 错误码

| 错误码 | 发生条件 | 前端提示 | 刷新 | 可重试 | 可能已成功 |
| --- | --- | --- | --- | --- | --- |
| `PRODUCT_NOT_FOUND` | ID不存在或不属于可信上下文 | 产品不存在。 | 是 | 否 | 否 |
| `PRODUCT_NOT_ACTIVE` | 产品已软删除 | 产品已被删除或不可用。 | 是 | 否 | 否 |
| `INVALID_PRODUCT_NAME` | 名称为空、过长或格式无效 | 请填写1至40字的产品名称。 | 否 | 修正后 | 否 |
| `INVALID_CATEGORY` | 分类为空或过长 | 请选择或填写有效分类。 | 否 | 修正后 | 否 |
| `INVALID_UNIT` | 单位为空或过长 | 请选择或填写有效单位。 | 否 | 修正后 | 否 |
| `INVALID_DESCRIPTION` | 介绍超过200字符 | 产品介绍不能超过200字。 | 否 | 修正后 | 否 |
| `INVALID_COVER` | 封面类型、字段、文件或token无效 | 产品封面无效，请重新选择。 | 否 | 是 | 否 |
| `INVALID_STOCK_QUANTITY` | 数量非整数、越界或调整无变化 | 请输入有效库存数量。 | 否 | 修正后 | 否 |
| `INVALID_MIN_STOCK` | 最低库存非负整数校验失败 | 请输入有效最低库存。 | 否 | 修正后 | 否 |
| `INSUFFICIENT_STOCK` | 事务内最新库存不足出库 | 库存不足，请刷新后重试。 | 是 | 刷新后 | 否 |
| `PRODUCT_HAS_STOCK` | 删除时stock大于0 | 请先将库存调整为0再删除。 | 是 | 处理后 | 否 |
| `PRODUCT_VERSION_CONFLICT` | 更新/删除版本落后 | 产品已被他人更新，请刷新后确认。 | 是 | 刷新后 | 否 |
| `INVENTORY_NOT_FOUND` | 未来余额模型中余额缺失；方案A通常不触发 | 库存数据不存在，请刷新。 | 是 | 否 | 否 |
| `STOCK_RECORD_NOT_FOUND` | 流水不存在或越权 | 库存记录不存在。 | 否 | 否 | 否 |
| `DUPLICATE_REQUEST` | 同requestKey使用不同业务输入 | 本次操作标识已被使用，请刷新确认。 | 是 | 不换key前否 | 是 |
| `FORBIDDEN` | 角色不足、越权字段或资源越界 | 当前账号无权执行此操作。 | 视情况 | 否 | 否 |
| `NO_ACTIVE_TEAM` | 无active成员关系 | 当前没有可用团队，请重新进入。 | 是 | 否 | 否 |
| `WAREHOUSE_NOT_FOUND` | 默认仓库不存在、disabled或越权 | 当前仓库不可用，请重新进入。 | 是 | 否 | 否 |
| `DATABASE_ERROR` | 已知数据库失败或事务冲突耗尽 | 数据处理失败，请稍后重试。 | 是 | 是，同key | 是 |
| `INTERNAL_ERROR` | 未归类异常 | 服务暂时不可用，请稍后重试。 | 否 | 是，同key | 是 |

数据库原始错误、堆栈、集合名、OPENID和文档内容不得返回前端。事务冲突可由服务端有限重试；耗尽后统一映射稳定错误码。

## 7. 权限与响应白名单

| action | owner | admin | viewer | 事务 | requestKey |
| --- | --- | --- | --- | --- | --- |
| `product.create` | 写 | 写 | 禁止 | 产品+initial流水 | 必填 |
| `product.list` | 读 | 读 | 读 | 否 | 否 |
| `product.detail` | 读 | 读 | 读 | 否 | 否 |
| `product.update` | 写 | 写 | 禁止 | 产品锁与版本校验 | 必填 |
| `product.delete` | 写 | 写 | 禁止 | 产品锁与软删除 | 必填 |
| `inventory.inbound` | 写 | 写 | 禁止 | 余额+流水 | 必填 |
| `inventory.outbound` | 写 | 写 | 禁止 | 余额+流水 | 必填 |
| `inventory.adjust` | 写 | 写 | 禁止 | 余额+流水 | 必填 |
| `stock.record.list` | 读 | 读 | 读 | 否 | 否 |
| `stock.record.detail` | 读 | 读 | 读 | 否 | 否 |

产品响应可以包含业务展示字段、库存字段、封面临时URL和时间。流水响应可以包含快照、前后库存、原因、来源去向、备注以及成员显示名。任何响应都不包含openId、内部userId、teamId、warehouseId、operatorId、createdBy、updatedBy、requestKey、inputHash或完整用户对象。
