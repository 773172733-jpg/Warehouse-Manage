# 阶段2C1接口契约

## 1. 通用约定

所有接口继续通过统一云函数 `warehouse-api` 调用。成功响应包含 `success:true`、`requestId`、`timestamp`和白名单 `data`；失败响应只返回稳定错误码和可展示中文信息，不返回数据库原始错误、堆栈或文档。

每个handler都使用可信OPENID重新查询active用户、成员、团队和仓库。客户端不得传入并被信任：`teamId`、`warehouseId`、`userId`、`operatorId`、`role`、`openId`、`createdBy`、`updatedBy`。发现这些字段返回 `FORBIDDEN`。

`productId`、`warehouseProductId`、`recordId`和 `operatorMemberId`只是候选资源定位字段，服务端仍须验证其归属。

### 1.1 分页

- 默认20条，最大50条。
- 共享目录按 `updatedAt desc, _id desc`；仓库产品按 `updatedAt desc, _id desc`；流水按 `createdAt desc, _id desc`。
- 游标由服务端编码，客户端原样回传。
- 无更多数据时 `nextCursor:null`；空列表返回 `items:[]`。
- 严禁一次性下载团队全部产品后本地筛选。

### 1.2 幂等

- 所有写接口要求8至64字符requestKey。
- 一次用户意图及其网络重试复用同一key；业务输入变化后生成新key。
- 服务端保存规范化输入摘要；product.create同key同输入返回首次结果，同key异参返回 `REQUEST_KEY_CONFLICT`；结构残缺或其他写接口冲突继续使用 `DUPLICATE_REQUEST`。
- `product.create`以 `teamId + requestKey`唯一；加入仓库以 `teamId + warehouseId + requestKey`唯一；库存流水以 `teamId + warehouseId + requestKey`唯一。

## 2. 组合响应模型

库存页和详情页不直接暴露两个集合，由warehouse-api返回组合对象：

```json
{
  "id": "product_xxx",
  "warehouseProductId": "warehouse_product_xxx",
  "name": "美纹纸胶带",
  "productCode": "MT-025",
  "category": "耗材",
  "unit": "卷",
  "brand": "3M",
  "specification": "25mm×50m",
  "description": "耐高温、不留残胶",
  "cover": {
    "type": "text",
    "text": "MT",
    "emoji": "",
    "background": "#EAF6EF",
    "url": ""
  },
  "stock": 8,
  "minStock": 15,
  "stockStatus": "low",
  "version": 2,
  "stockVersion": 4,
  "updatedAt": "server-date"
}
```

products提供产品资料；warehouse_products提供当前仓库实例和库存。快照落后时，响应必须使用products权威字段并触发安全修复。

## 3. 产品接口

### 3.1 product.create

权限：owner、admin。

请求：

```json
{
  "name": "800×800浅灰柔光砖",
  "productCode": "GZ-800-H",
  "category": "瓷砖",
  "unit": "箱",
  "brand": "冠珠",
  "specification": "800×800mm",
  "description": "浅灰色柔光砖",
  "coverType": "text",
  "coverText": "柔光砖",
  "coverEmoji": "",
  "coverBackground": "#EAF6EF",
  "initialStock": 126,
  "minStock": 30,
  "requestKey": "product_create_xxx"
}
```

客户端禁止提交以下字段，这些字段只能由 warehouse-api 服务端生成：

- `searchKeywords`：由服务端从 name、productCode、category、brand、specification 统一生成
- `normalizedName`：服务端 NFKC 规范化
- `normalizedCode`：服务端规范化编号

即使请求中出现上述字段，接口也必须丢弃并以服务端生成为准。

事务必须创建products、默认仓库warehouse_products，并在initialStock大于0时创建initial流水，同时更新teams.activeProductCount。任一步失败全部回滚。产品名称和编号允许重复；activeProductCount达到99,999时返回 `PRODUCT_LIMIT_REACHED`。

客户端不能提交version、productVersion、stockVersion、stockStatus或身份字段。

返回：

```json
{
  "product": {
    "id": "product_xxx",
    "name": "800×800浅灰柔光砖",
    "productCode": "GZ-800-H",
    "category": "瓷砖",
    "unit": "箱",
    "version": 1
  },
  "warehouseProduct": {
    "id": "warehouse_product_xxx",
    "productId": "product_xxx",
    "stock": 126,
    "minStock": 30,
    "stockStatus": "normal",
    "stockVersion": 1
  },
  "initialRecord": {
    "id": "record_xxx",
    "type": "initial",
    "changeQuantity": 126,
    "beforeStock": 0,
    "afterStock": 126
  },
  "idempotent": false
}
```

### 3.2 product.list

权限：owner、admin、viewer。

请求：

```json
{
  "keyword": "柔光",
  "category": "瓷砖",
  "stockStatus": "low",
  "pageSize": 20,
  "cursor": "opaque_cursor"
}
```

行为：

- 无关键词时从当前仓库warehouse_products按库存条件和快照分页。
- 2C2A使用warehouse_products受控快照执行标准化名称前缀、编号精确或受控关键词搜索；products仍是权威主资料，product.update上线后由版本同步维护快照。
- 不承诺99,999规模下任意包含式正则高性能。
- 默认只返回active products和active warehouse_products。
- 2C2A不返回summary，统计延后到2C5。

返回 `items`、`nextCursor`、`hasMore`和 `pageSize`。按 `updatedAt desc, _id desc` 使用不透明游标，不使用无限增长offset；空仓库或无匹配项返回空数组。

### 3.3 product.detail

权限：owner、admin、viewer。

请求：

```json
{
  "productId": "product_xxx"
}
```

也可只传 `warehouseProductId`，但两个标识必须且只能提供一个。服务端读取products和当前仓库warehouse_products，返回 `product`、`warehouseProduct`和权限标记；2C2A不返回最近流水。products deleted返回 `PRODUCT_NOT_ACTIVE`；当前仓库实例不存在或removed返回 `PRODUCT_NOT_IN_WAREHOUSE`。

### 3.4 product.update

权限：owner、admin。修改团队共享主资料。

```json
{
  "productId": "product_xxx",
  "version": 2,
  "name": "美纹纸胶带 25mm",
  "productCode": "MT-025",
  "category": "耗材",
  "unit": "卷",
  "brand": "3M",
  "specification": "25mm×50m",
  "description": "更新后的介绍",
  "cover": {
    "type": "text",
    "text": "MT",
    "emoji": "",
    "assetKey": "",
    "fileId": "",
    "background": "#EAF6EF",
    "uploadToken": ""
  },
  "requestKey": "product_update_xxx"
}
```

客户端禁止提交以下字段，这些字段只能由 warehouse-api 服务端生成：

- `searchKeywords`：编辑后由服务端从最新 name、productCode、category、brand、specification 重新生成
- `normalizedName`：服务端 NFKC 规范化
- `normalizedCode`：服务端规范化编号

即使请求中出现上述字段，接口也必须丢弃并以服务端生成为准。

请求禁止出现stock、minStock、stockStatus、stockVersion。version不匹配返回 `PRODUCT_VERSION_CONFLICT`。V1事务同步当前仓库快照；多仓超出事务能力时异步补偿，读接口以products为准。

### 3.5 product.removeFromWarehouse

权限：owner、admin。V1“删除产品”的真实含义。

```json
{
  "warehouseProductId": "warehouse_product_xxx",
  "reason": "当前仓库不再使用",
  "requestKey": "warehouse_remove_xxx"
}
```

事务内要求当前仓库实例stock为0，将status改为removed并递减products.activeWarehouseCount。历史流水和共享主图保留。stock大于0返回 `PRODUCT_HAS_STOCK`。

### 3.6 product.restoreToWarehouse

权限：owner、admin。V1必须提供回收站恢复入口。

```json
{
  "warehouseProductId": "warehouse_product_xxx",
  "requestKey": "warehouse_restore_xxx"
}
```

复用原warehouse_products，stock保持0，复制最新主数据快照并递增products.activeWarehouseCount。共享目录产品必须active。

### 3.7 product.catalog.delete

权限：仅owner。请求只允许：

```json
{
  "productId": "product_xxx",
  "expectedVersion": 4,
  "reason": "停止维护该共享产品",
  "requestKey": "catalog_delete_xxx"
}
```

服务端先读取products做快速失败检查，正式事务内重新锁定products并要求status为active、version匹配、activeWarehouseCount为非负安全整数且等于0。product.create、仓库移除、仓库恢复以及未来新增仓库实例都必须在同一事务内更新这条products记录，因此目录删除与仓库激活写入会形成事务冲突互斥。成功后products软删除、version加1，teams.activeProductCount减1；warehouse_products和stock_records不变。接口不执行跨仓扫描，也不需要新增warehouse_products索引。

### 3.8 product.catalog.deleted.list

权限：仅owner。请求允许keyword、category、cursor、pageSize。只返回可信当前团队的deleted products，默认20、最大50，按 `updatedAt desc, _id desc` 使用不透明游标。响应包含安全version、activeWarehouseCount、canRestore和展示字段，不返回身份或幂等字段。

### 3.9 product.catalog.restore

权限：仅owner。请求：

```json
{
  "productId": "product_xxx",
  "expectedVersion": 5,
  "requestKey": "catalog_restore_xxx"
}
```

事务要求products为deleted、version匹配、activeWarehouseCount仍为0且团队activeProductCount未达99,999。成功后复用原productId、status改回active、version加1、团队计数加1。接口不创建或恢复warehouse_products，不创建流水；用户需在产品回收站手动恢复原仓库实例。

`product.catalog.list` 和 `product.addToWarehouse` 仍为未来多仓目录能力。

## 4. 库存接口

所有库存接口操作当前可信仓库的warehouse_products。owner/admin可写，viewer禁止。余额更新和流水创建必须在同一事务。

### 4.1 inventory.inbound

```json
{
  "productId": "product_xxx",
  "warehouseProductId": "warehouse_product_xxx",
  "quantity": 20,
  "sourceOrDestination": "佛山供应商",
  "remark": "七月补货",
  "requestKey": "inventory_inbound_xxx"
}
```

quantity为正整数，流水changeQuantity为正。

### 4.2 inventory.outbound

请求字段与入库相同。事务内按最新stock再次校验；库存不足返回 `INSUFFICIENT_STOCK`，余额和流水都不写入。流水changeQuantity为负。

### 4.3 inventory.adjust

```json
{
  "productId": "product_xxx",
  "warehouseProductId": "warehouse_product_xxx",
  "targetStock": 18,
  "reason": "盘点修正",
  "remark": "补录未登记到货",
  "requestKey": "inventory_adjust_xxx"
}
```

targetStock为非负整数；服务端计算changeQuantity。目标等于当前库存时拒绝，不创建零变化流水。

### 4.4 inventory.threshold.update

权限：owner、admin。修改仓库级最低库存，不修改products。

```json
{
  "productId": "product_xxx",
  "warehouseProductId": "warehouse_product_xxx",
  "minStock": 12,
  "expectedStockVersion": 4,
  "requestKey": "threshold_update_xxx"
}
```

事务内更新minStock、重算stockStatus并递增stockVersion。版本不匹配返回 `STOCK_VERSION_CONFLICT`。

库存写统一返回最新stock、minStock、stockStatus、stockVersion以及新流水；阈值修改不创建库存流水，可写审计日志。

## 5. 流水接口

### 5.1 stock.record.list

权限：owner、admin、viewer。

```json
{
  "keyword": "滨江",
  "productId": "product_xxx",
  "warehouseProductId": "warehouse_product_xxx",
  "type": "outbound",
  "operatorMemberId": "member_xxx",
  "dateFrom": "2026-07-01T00:00:00.000Z",
  "dateTo": "2026-08-01T00:00:00.000Z",
  "pageSize": 20,
  "cursor": "opaque_cursor"
}
```

客户端不能传operatorId。服务端验证operatorMemberId属于当前团队后映射查询。产品、仓库实例或成员离开后，授权用户仍可读取历史快照。空状态返回items空数组。

### 5.2 stock.record.detail

请求只传recordId。服务端验证teamId和warehouseId归属，返回产品/单位/操作人快照、前后库存、原因、来源去向、备注和时间。流水没有update或delete接口。

## 6. 图片辅助接口

- `product.cover.prepare`：owner/admin提交文件元数据和requestKey，云端限制JPEG/PNG/WebP、最大2MB并返回短期staging路径和uploadToken。
- `product.cover.discard`：清理未绑定staging文件。

客户端上传前先校验，云端绑定前再次校验。数据库只保存fileID，响应可返回临时URL。更新旧图延迟回收；仓库移除和全局软删除都不立即删除共享主图。

## 7. 错误码

| 错误码 | 条件 | 前端提示 | 刷新/重试 |
| --- | --- | --- | --- |
| `PRODUCT_NOT_FOUND` | 产品不存在或越权 | 产品不存在。 | 不重试 |
| `PRODUCT_NOT_ACTIVE` | 共享目录产品deleted | 产品已被删除。 | 刷新 |
| `PRODUCT_LIMIT_REACHED` | active目录产品已达99,999 | 团队产品数量已达上限。 | 不重试 |
| `PRODUCT_NOT_IN_WAREHOUSE` | 当前仓库实例不存在或removed | 当前仓库没有该产品。 | 刷新 |
| `REQUEST_KEY_CONFLICT` | product.create同key异参 | 提交标识已用于其他内容。 | 更换意图后生成新key |
| `INVALID_CURSOR` | 游标结构或字段无效 | 分页状态已失效。 | 从首屏重载 |
| `INVALID_PAGE_SIZE` | 每页不在1至50 | 每页数量无效。 | 修正输入 |
| `INVALID_PRODUCT_NAME` | 名称无效 | 请填写1至40字产品名称。 | 修正输入 |
| `INVALID_CATEGORY` | 分类无效 | 请选择或填写有效分类。 | 修正输入 |
| `INVALID_UNIT` | 单位无效 | 请选择或填写有效单位。 | 修正输入 |
| `INVALID_DESCRIPTION` | 介绍超过200字 | 产品介绍不能超过200字。 | 修正输入 |
| `INVALID_COVER` | 封面、文件或token无效 | 请重新选择产品封面。 | 可重试 |
| `INVALID_STOCK_QUANTITY` | 数量无效或调整无变化 | 请输入有效库存数量。 | 修正输入 |
| `INVALID_MIN_STOCK` | 阈值无效 | 请输入有效最低库存。 | 修正输入 |
| `WAREHOUSE_PRODUCT_NOT_FOUND` | 仓库实例不存在或越权 | 当前仓库没有该产品。 | 刷新 |
| `WAREHOUSE_PRODUCT_NOT_ACTIVE` | 仓库实例removed | 产品已从当前仓库移除。 | 刷新 |
| `PRODUCT_HAS_STOCK` | 非零库存尝试移除 | 当前库存不为0，请先完成出库或库存调整。 | 处理后重试 |
| `PRODUCT_ALREADY_REMOVED` | 仓库实例已经removed且非原请求重试 | 该产品已经从当前仓库移除。 | 刷新 |
| `PRODUCT_ALREADY_DELETED` | 目录产品已经deleted且非原请求重试 | 该共享产品目录已经删除。 | 刷新 |
| `PRODUCT_ALREADY_ACTIVE` | 仓库实例或目录已经active且非原恢复重试 | 该产品已经恢复。 | 刷新 |
| `PRODUCT_STILL_IN_WAREHOUSE` | activeWarehouseCount大于0 | 该产品仍存在于仓库中，请先从所有仓库移除。 | 处理后重试 |
| `PRODUCT_WAREHOUSE_STATE_CONFLICT` | 计数与仓库实例状态矛盾 | 产品仓库状态异常，请刷新后重试。 | 检查数据 |
| `PRODUCT_CATALOG_DELETED` | 共享产品目录非active | 共享产品目录已删除，暂时无法恢复。 | 不重试 |
| `INVALID_PRODUCT_VERSION` | expectedVersion不是正整数 | 产品版本无效，请刷新。 | 刷新 |
| `INVALID_DELETE_REASON` | 删除原因超过100字符 | 删除原因不能超过100个字符。 | 修正输入 |
| `INSUFFICIENT_STOCK` | 最新库存不足 | 库存不足，请刷新。 | 刷新后重试 |
| `PRODUCT_VERSION_CONFLICT` | 主资料版本落后 | 产品已被更新，请刷新确认。 | 刷新后重试 |
| `STOCK_VERSION_CONFLICT` | 仓库余额版本落后 | 库存已变化，请刷新。 | 刷新后重试 |
| `STOCK_RECORD_NOT_FOUND` | 流水不存在或越权 | 库存记录不存在。 | 不重试 |
| `DUPLICATE_REQUEST` | 同key异参 | 操作标识已使用，请刷新确认。 | 可能已成功 |
| `FORBIDDEN` | 角色不足或禁传字段 | 当前账号无权执行。 | 不重试 |
| `NO_ACTIVE_TEAM` | 无active团队 | 当前没有可用团队。 | 重新进入 |
| `WAREHOUSE_NOT_FOUND` | 仓库无效 | 当前仓库不可用。 | 重新进入 |
| `DATABASE_ERROR` | 数据库或事务冲突耗尽 | 数据处理失败，请稍后重试。 | 原key重试，可能已成功 |
| `INTERNAL_ERROR` | 未归类异常 | 服务暂时不可用。 | 原key重试，可能已成功 |

## 8. 权限与事务矩阵

| action | owner | admin | viewer | 事务 | requestKey |
| --- | --- | --- | --- | --- | --- |
| `product.create` | 写 | 写 | 禁止 | products+warehouse_products+initial | 必填 |
| `product.list/detail` | 读 | 读 | 读 | 否 | 否 |
| `product.update` | 写 | 写 | 禁止 | 主数据+快照 | 必填 |
| `product.removeFromWarehouse` | 写 | 写 | 禁止 | 仓库实例+产品计数 | 必填 |
| `product.restoreToWarehouse` | 写 | 写 | 禁止 | 仓库实例+产品计数 | 必填 |
| `product.removed.list` | 读 | 读 | 禁止 | 否 | 否 |
| `product.catalog.delete/restore` | 写 | 禁止 | 禁止 | 产品+团队计数 | 必填 |
| `product.catalog.deleted.list` | 读 | 禁止 | 禁止 | 否 | 否 |
| `inventory.inbound/outbound/adjust` | 写 | 写 | 禁止 | 余额+流水 | 必填 |
| `inventory.threshold.update` | 写 | 写 | 禁止 | 仓库实例 | 必填 |
| `stock.record.list/detail` | 读 | 读 | 读 | 否 | 否 |

响应使用字段白名单，不返回openId、内部userId、teamId、warehouseId、operatorId、createdBy、updatedBy、requestKey、inputHash或完整数据库文档。
