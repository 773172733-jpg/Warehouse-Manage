# 阶段2A部署与验收

## 部署前阻断检查

在微信开发者工具中确认：

- 项目 AppID 是 `wxd5819a772c90b7a2`。
- 当前 CloudBase 环境 ID 是 `cloud1-d8gm59cz2be4e7c23`。
- 待部署目录是本项目的 `cloudfunctions/warehouse-api`。

任何一项不一致都立即停止。不要在其他项目环境中创建集合、索引、权限或云函数。

## 1. 创建集合

打开“云开发” -> “数据库”，在轻仓环境依次创建：

1. `users`
2. `teams`
3. `team_members`
4. `warehouses`

不要手工插入示例数据。字段由 `warehouse-api` 首次调用写入，结构见 [`../database/collections.md`](../database/collections.md)。

## 2. 创建索引

进入每个集合的“索引管理”，按 [`../database/indexes.md`](../database/indexes.md) 创建全部阶段2A索引。必须重点核对三个唯一索引：

- `users.openId`
- `teams.ownerId + createRequestKey`
- `team_members.teamId + userId`

等待索引状态变为可用后再继续。索引创建失败或仍在构建时，不执行真实 bootstrap 和团队创建。

## 3. 设置权限

进入四个集合各自的“权限管理”，切换为自定义安全规则，保存：

```json
{
  "read": false,
  "write": false
}
```

四个集合都要设置。不要使用“所有用户可读写”。详细原则见 [`../database/permissions.md`](../database/permissions.md)。

## 4. 部署云函数

在项目树右键 `cloudfunctions/warehouse-api`，选择“上传并部署：云端安装依赖”。部署完成后在轻仓环境云函数列表确认：

- 函数名为 `warehouse-api`
- 状态正常
- 最近更新时间与本次部署一致
- 依赖 `wx-server-sdk@4.0.2` 安装成功

## 5. 接口验证

先在微信开发者工具编译 startup 页面。也可在云函数控制台测试，事件格式如下：

```json
{
  "action": "system.ping",
  "data": {},
  "requestId": "manual_ping_001"
}
```

`system.ping` 应返回 `success: true`。`user.bootstrap` 必须从真实小程序调用，以便 `cloud.getWXContext()` 获得当前微信身份；首次调用后应进入创建团队页，不要求昵称头像授权。

## 6. 首次创建团队

1. 输入2至30字符的团队名称。
2. 保留或修改“默认仓库”，名称为1至30字符。
3. 连续快速点击创建按钮，确认只生成一个团队。
4. 在网络失败后重试同一次输入，确认仍只有一个团队。
5. 创建成功后应使用 `wx.switchTab` 进入库存页。
6. 重新编译小程序，应直接进入库存页，不再显示创建页。

## 7. 核对四个集合

按当前测试用户核对：

- `users`：一条用户记录，`currentTeamId` 和 `currentWarehouseId` 已填写。
- `teams`：一条 active 团队，`ownerId` 对应该用户，`defaultWarehouseId` 已填写。
- `team_members`：一条 active owner 关系。
- `warehouses`：一条 active 默认仓库，`isDefault` 为 `true`。

不要在截图、日志或回执中展示完整 `openId`。快速点击和重复 bootstrap 后，上述四个集合不应出现重复记录。

## 8. 隔离核对

回到云开发控制台环境选择器，再次确认当前环境 ID 是 `cloud1-d8gm59cz2be4e7c23`。检查其他环境时只做只读确认：不应出现本次创建的轻仓测试团队，不要修改或清理其他环境。

## 9. 清理测试团队

阶段2A未提供删除接口。确需重测时，仅在轻仓环境控制台按以下顺序人工清理：

1. 记录测试用户、团队和仓库的文档 ID。
2. 删除对应 `team_members` 文档。
3. 删除对应 `warehouses` 文档。
4. 删除对应 `teams` 文档。
5. 将对应 `users.currentTeamId` 和 `currentWarehouseId` 改为空字符串。

通常保留 `users` 文档即可重新测试团队创建。若删除用户文档，下一次启动会重新创建。操作前确认筛选条件只命中测试数据，禁止批量清空集合。

## 手动验收清单

- 首次启动创建一条用户，不白屏、不无限重试。
- 无团队进入创建团队页。
- 输入校验、失败保留输入和加入入口占位正常。
- 创建成功进入库存 tab。
- 重启后直接进入库存 tab。
- 重复提交和快速连点不产生重复团队。
- 四个集合的客户端直接读写均被拒绝。
- 控制台日志含 requestId，但不含完整 OPENID 和用户对象。
- 阶段1原型页面可继续浏览，但其业务数据仍为本地模拟数据。
