# 阶段2C2B2部署与验收

## 部署前确认

本阶段只更新小程序端页面和本地测试。若阶段2C2A的 `warehouse-api` 已部署，且 `products`、`warehouse_products`、`stock_records` 及必需索引已经生效，则无需重新部署云函数，也无需新增集合、索引或权限。

在微信开发者工具确认正式AppID、轻仓独立CloudBase环境和 `warehouse-api` 后，从startup正常编译。不要使用游客模式做真实权限或数据库验收。

## 创建后立即显示

1. 使用owner或admin进入库存页。
2. 记录当前“已加载产品”数量。
3. 点击新增，完成三步表单并真实创建一个具有明显名称的产品。
4. 创建成功自动返回库存Tab。
5. 确认首页重新加载，并显示刚创建的产品。
6. 点击该卡片，确认详情名称、编号、分类、单位、封面和库存与创建内容一致。
7. 注意“已加载产品”只代表当前已加载结果，不是全仓库准确总数。

若云端集合已有新增记录但首页不显示，先清除搜索和筛选，再下拉刷新；随后在Network或云函数日志检查 `product.list` 的业务错误，不要用mock兜底。

## 搜索和筛选

1. 输入产品名称前缀，停顿约半秒，确认列表从云端刷新。
2. 使用完整产品编号搜索，确认能够命中。
3. 清空搜索词，确认恢复第一页。
4. 分别选择固定分类、正常、低库存和缺货。
5. 组合分类与库存状态，确认结果符合云端数据。
6. 快速连续更换搜索词，确认旧结果不会在稍后覆盖新结果。

搜索只保证名称前缀、编号和受控关键词能力，不保证任意全文包含。

## 分页验收

分页测试需要当前筛选条件下至少21个产品：

1. 首次进入确认只读取20条，不一次读取全部产品。
2. 滚动到底部，确认使用后端 `nextCursor` 加载下一页。
3. 检查跨页没有重复卡片，顺序保持最近更新优先。
4. 到达最后一页后继续滚动，确认不再发请求。
5. 下拉刷新后确认cursor被清空并重新读取第一页。
6. 切换搜索、分类或库存状态后，确认旧cursor不会被复用。

CloudBase控制台若报告缺少索引，应对照 `database/indexes.md` 检查阶段2C2A必需组合索引状态；不要通过前端全量加载绕开索引。

## 权限验收

owner/admin：

- 显示新增产品按钮。
- 能读取真实列表和详情。
- 详情显示入库、出库和更多入口，但点击只显示后续阶段提示，不修改数据。

viewer：

- 不显示新增产品按钮。
- 能读取真实列表和详情。
- 详情不显示编辑、库存操作或移除入口。

pending、removed或无团队账号应被重新引导到startup，不得继续显示旧团队产品。

## 状态和错误验收

1. 临时断网后进入库存页，确认显示错误和重新加载按钮，不出现mock产品。
2. 恢复网络并重试，确认真实数据出现。
3. 使用不存在的 `warehouseProductId` 编译详情页，确认显示“产品不存在或已被移除”。
4. 使用非法或空ID编译详情页，确认明确提示且不白屏。
5. 下拉刷新时确认页面不会出现重复请求或重复卡片。
6. 检查详情的库存记录区域只显示后续接入占位，不显示历史mock流水。

## 当前边界

- 产品编辑将在2C3A接入。
- 当前仓库移除将在2C3B接入。
- 图片上传将在2C3C接入。
- 入库、出库和调整将在2C4接入。
- 全局准确统计和分类管理安排在2C5。
