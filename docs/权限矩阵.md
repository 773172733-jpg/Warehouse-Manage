# 权限矩阵

## owner

- 团队创建者
- 管理管理员
- 管理普通成员
- 管理共享产品、仓库实例和库存
- 全局删除与恢复共享目录产品
- 团队最高权限

## admin

- 新增和编辑产品
- 入库
- 出库
- 调整库存
- 从当前仓库移除和恢复零库存产品
- 查看记录

## viewer

- 查看产品
- 搜索产品
- 查看库存
- 不允许任何写操作

## 权限控制原则

前端隐藏按钮不等于权限控制。所有写操作必须由云函数查询 `team_members` 验证，不允许信任前端传入的 `role`、`teamId`、`openId` 或 `userId`。

产品回收站表示软移除或恢复当前 `warehouse_products`，owner/admin可操作；共享目录回收站表示软删除或恢复团队 `products`，仅owner可操作。目录恢复不会自动恢复仓库实例，两层操作都永久保留stock_records。

阶段2C3B已在云端实际实现：owner可调用全部产品接口和 `product.catalog.delete/deleted.list/restore`；admin可调用仓库级创建、编辑、移除、回收站列表和恢复，但不能访问目录管理接口；viewer只能调用 `product.list/detail`；pending/removed成员不能读取产品数据。直接输入页面路径也会再次校验角色，最终权限始终由warehouse-api校验。
