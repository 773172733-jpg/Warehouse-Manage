# 数据模型

阶段2C1已定稿产品、库存和流水V1模型。本文件是集合总览；完整字段、事务、索引和迁移依据见：

- `docs/阶段2C1产品库存与流水架构.md`
- `docs/阶段2C1接口契约.md`
- `docs/阶段2C1实施拆分与迁移.md`

2C1只更新文档，不创建真实产品或流水数据。

## 集合

- `users`
- `teams`
- `team_members`
- `warehouses`
- `products`
- `warehouse_products`
- `stock_records`
- `categories`
- `invites`
- `audit_logs`

所有团队业务集合需要预留 `teamId`。仓库业务集合需要预留 `warehouseId`。

## 共享产品目录

`products`是团队级权威主资料，保存teamId、名称、编号、分类、单位、介绍、搜索关键词、共享封面、version、activeWarehouseCount、幂等和删除恢复审计字段。

products不得保存warehouseId、stock、minStock、stockStatus或stockVersion。名称允许重名，以 `_id` 为唯一身份。每团队最多99,999个active产品。

## 仓库产品与库存余额

`warehouse_products`保存共享产品在具体仓库的实例：

- 唯一关系：`teamId`、`warehouseId`、`productId`
- 余额：`stock`、`minStock`、`stockStatus`、`stockVersion`
- 快照：`productVersion`和首页必要产品展示快照
- 生命周期：active/removed、创建/更新/移除/恢复及幂等字段

stockStatus由云端计算并持久化。移除要求stock为0，恢复复用原记录且stock保持0。products始终是主资料权威来源。

## 库存流水

`stock_records`保存：

- 归属与产品：`teamId`、`warehouseId`、`productId`、`warehouseProductId`
- 快照：产品名称、编号、单位和操作人显示名
- `type`：`initial`、`inbound`、`outbound`、`adjust`
- `changeQuantity`、`beforeStock`、`afterStock`
- `reason`、`sourceOrDestination`、`remark`
- 可信操作人字段、`requestAction`、`requestKey`、输入摘要和 `createdAt`

流水永久保留，用户不能修改或删除。库存变化与流水创建必须在 `warehouse-api` 的同一事务完成，并使用requestKey防止重复提交；未来只允许不破坏审计历史的冷归档。
