# 数据模型

阶段0只规划模型，不创建真实数据。

## 集合

- `users`
- `teams`
- `team_members`
- `warehouses`
- `products`
- `stock_records`
- `categories`
- `invites`
- `audit_logs`

所有团队业务集合需要预留 `teamId`。仓库业务集合需要预留 `warehouseId`。

## 产品

`products` 规划字段：

- `name`
- `code`
- `description`
- `categoryId`
- `unit`
- `stock`
- `minStock`
- `coverMode`
- `systemAssetKey`
- `customFileId`
- `displayText`
- `searchText`
- `searchKeywords`
- `status`
- `createdBy`
- `updatedBy`
- `createdAt`
- `updatedAt`
- `deletedAt`
- `deletedBy`

产品使用软删除，删除时写入 `deletedAt` 和 `deletedBy`，不直接物理删除。

## 库存流水

`stock_records` 规划字段：

- `type`：`initial`、`inbound`、`outbound`、`adjustment`
- `quantityDelta`
- `stockBefore`
- `stockAfter`
- `reason`
- `remark`
- `operatorId`
- `requestKey`
- `createdAt`

库存不允许仅修改 `products.stock` 而不产生 `stock_records` 流水。后续库存变更必须在云函数中完成，并考虑并发一致性与重复提交。
