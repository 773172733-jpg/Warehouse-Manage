# 数据模型

阶段2C1已定稿产品、库存和流水V1模型。本文件是集合总览；完整字段、事务、索引和迁移依据见：

- `docs/阶段2C1产品库存与流水架构.md`
- `docs/阶段2C1接口契约.md`
- `docs/阶段2C1实施拆分与迁移.md`

2C1只更新文档，不创建真实产品或流水数据。

## 集合

- `users`
- `teams`
- `team_members`
- `warehouses`
- `products`
- `stock_records`
- `categories`
- `invites`
- `audit_logs`

所有团队业务集合需要预留 `teamId`。仓库业务集合需要预留 `warehouseId`。

## 产品与库存余额

V1采用仓库级 `products` 文档，不创建单独的余额集合。每个文档同时保存：

- 归属：`teamId`、`warehouseId`
- 产品资料：`name`、`normalizedName`、`code`、`category`、`unit`、`brand`、`specification`、`description`、`keywords`、`searchText`
- 封面：`coverType`、`coverText`、`coverEmoji`、`coverAssetKey`、`coverFileId`、`coverBackground`
- 余额：`stock`、`minStock`、`stockStatus`、`stockVersion`
- 生命周期与并发：`status`、`version`、`createRequestKey`、创建/更新/删除审计字段

`stockStatus`由云端按统一规则计算并持久化。产品使用软删除；有库存时禁止删除。普通产品编辑不得修改stock。

## 库存流水

`stock_records`保存：

- 归属与产品：`teamId`、`warehouseId`、`productId`
- 快照：产品名称、编号、单位和操作人显示名
- `type`：`initial`、`inbound`、`outbound`、`adjust`
- `changeQuantity`、`beforeStock`、`afterStock`
- `reason`、`sourceOrDestination`、`remark`
- 可信操作人字段、`requestAction`、`requestKey`、输入摘要和 `createdAt`

流水创建后不修改、不物理删除。库存变化与流水创建必须在 `warehouse-api` 的同一事务完成，并使用requestKey防止重复提交。
