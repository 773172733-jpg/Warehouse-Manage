# 阶段2C3A产品编辑与仓库回收站

## 范围

本阶段实现团队级产品主资料编辑，以及产品在当前可信仓库中的软移除、回收站查询和原实例恢复。阶段2C3B已在此基础上另行实现共享目录删除与恢复；库存写入、最低库存修改、图片上传和真实流水列表仍未实现。

## 云端接口

### product.update

owner/admin提交 `productId`、正整数 `expectedVersion`、产品主资料白名单和 `requestKey`。服务端在事务内验证active成员、团队、仓库、products及active warehouse_products，并执行：

1. 比较products.version；冲突返回 `PRODUCT_VERSION_CONFLICT`。
2. 服务端重新生成normalizedName、normalizedCode和searchKeywords。
3. products.version只增加1，写入可信updatedBy与服务端时间。
4. 同事务同步当前warehouse_products展示快照和productVersion。
5. stock、minStock、stockStatus、stockVersion及stock_records均不变。

`lastUpdateRequestKey/hash/resultVersion` 保存最后一次更新意图。同键同参在状态未被后续更新取代时返回 `idempotent=true` 且不再次递增version；同键异参返回 `REQUEST_KEY_CONFLICT`。

### product.removeFromWarehouse

owner/admin只提交 `warehouseProductId`、可选reason和requestKey。事务要求当前实例active且stock严格等于0；否则返回 `PRODUCT_HAS_STOCK`。成功后实例status改为removed，保留原ID、stock=0、minStock、stockVersion、快照和全部流水，并将products.activeWarehouseCount减1且不低于0。products.status和teams.activeProductCount不变。

### product.removed.list

仅owner/admin可读。接口只查询可信当前仓库的removed实例，支持keyword、category、20至50条pageSize和稳定cursor。排序使用updatedAt与_id，复用现有索引。响应将warehouse_products移除信息与products权威主资料安全组合，不返回身份或幂等字段。

### product.restoreToWarehouse

owner/admin只提交原warehouseProductId和requestKey。事务要求实例removed、stock=0且共享products仍active。恢复复用原记录，以products最新字段刷新快照，status改为active，stock保持0、stockStatus为out、stockVersion不变，并将products.activeWarehouseCount加1。teams.activeProductCount和stock_records不变。

## 页面流程

- 产品详情的“编辑产品”进入原product-edit页面edit模式，真实读取product.detail并保存product.id/version。
- create模式仍保留三步创建；edit模式库存步骤为只读说明，更新payload不含initialStock或minStock。
- version冲突时不覆盖，用户只能重新加载最新内容或取消返回。
- 产品详情仅在stock=0时确认移除；云端仍做最终校验。
- “我的”页owner/admin显示回收站入口，viewer不显示且直接进入也会被拦截；库存首页保持高频库存操作精简布局。
- 恢复成功后回收站重新查询第一页；库存首页下次显示时重新查询真实列表。

阶段2C3B后，“产品回收站”专指当前仓库已移除的warehouse_products。owner在后端返回canDeleteCatalog=true时还可发起团队级目录软删除；admin只能恢复仓库实例。团队级deleted products位于独立“共享目录回收站”，恢复目录后仍需回到产品回收站手动恢复原仓库实例。

页面不写Storage，不缓存完整产品对象，不直接调用wx.cloud或数据库。网络失败保留当前写意图requestKey，成功、异参冲突或取消意图后清除。

## 图片边界

新建和主动替换仍只支持none、text、emoji。若历史数据已有image封面，edit模式可不提交封面字段以保留原图；客户端不能提交本地路径或任意fileID。图片上传和替换属于2C3C。
