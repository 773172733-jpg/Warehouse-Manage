# 阶段2C2B2真实库存首页与产品详情

## 目标

阶段2C2B2将库存首页和产品详情页从阶段1的本地mock切换到真实云端数据。页面只通过 `product-service.js` 调用 `product.list` 和 `product.detail`，不直接调用 `wx.cloud` 或数据库。

旧 `mock-data.js` 文件仍保留给历史阶段参考，但库存首页和详情页已经解除引用；云端失败时只显示错误和重试，不会使用mock冒充成功。

## 库存首页

库存首页调用：

```text
productService.listProducts({
  keyword,
  category,
  stockStatus,
  cursor,
  pageSize: 20,
  sort: 'updated_desc'
})
```

空白的搜索词、全部分类和全部库存状态不会传给云端。团队、仓库、用户、角色和openId完全由云函数可信上下文确定，页面不会提交这些字段。

首次加载、筛选变化、搜索变化和下拉刷新都会读取第一页。页面使用后端返回的 `nextCursor` 和 `hasMore` 加载下一页，不使用offset、不解析cursor，也不一次加载全部产品。按 `warehouseProductId` 合并去重，避免分页边界出现重复卡片。

每次重置查询都会递增请求版本。旧搜索或旧筛选响应即使更晚返回，也不能覆盖新结果。加载更多时如果cursor失效，页面会清空cursor并重新读取第一页，不循环自动重试。

## 搜索、分类和库存状态

搜索输入使用400毫秒防抖，实际能力遵循云端契约：

- 编号规范化精确匹配。
- 产品名称规范化前缀匹配。
- 云函数生成的受控关键词匹配。

页面不在客户端对当前数组做假模糊搜索，也不承诺description全文包含搜索。

分类复用当前产品创建页的固定分类范围：瓷砖、工具、五金、耗材、办公用品、其他。“全部”不传category。分类变化会重新查询云端第一页，不只过滤已加载数组。统一分类管理仍可在2C5扩展。

库存状态支持全部、正常、低库存和缺货，分别对应空值、`normal`、`low`、`out`。状态显示使用云端返回字段；未知值安全显示为“状态未知”，不会导致页面白屏。

当前后端只支持 `updated_desc`，因此页面不提供名称或库存本地排序，避免把单页排序伪装成全量排序。

## 已加载统计

`product.list` 当前不返回全仓库准确统计。首页保留原摘要布局，但文案明确为：

- 已加载产品
- 已加载低库存
- 已加载缺货

数字只统计当前查询条件下已加载到页面的真实items。页面不会扫描全部分页或声称这是全仓库总数；全局准确统计安排在2C5。

## 产品卡片与封面

产品卡片来自 `warehouse_products` 的白名单快照，显示名称、编号、分类、单位、品牌或规格、库存、库存状态和封面。详情路由只传经过URL编码的真实 `warehouseProductId`：

```text
/pages/product-detail/product-detail?warehouseProductId=...
```

文字封面显示受控文字，emoji封面显示受控emoji，none或未知封面安全降级为默认文字占位。数量异常显示“—”，内部ID、版本、审计和幂等字段均不展示。

## 产品详情

详情页调用：

```text
productService.getProductDetail({ warehouseProductId })
```

`products` 响应提供权威名称、编号、分类、单位、品牌、规格、介绍和封面；`warehouse_products` 响应提供当前仓库的库存、最低库存、库存状态和更新时间；`permissions` 控制写操作入口。

旧 `id` 参数暂时兼容为 `warehouseProductId` 别名，但不再用于mock数组查找。缺失或非法ID显示明确错误，不会白屏。

owner和admin可以看到新增入口及详情页操作入口；viewer可读取列表和详情，但看不到写入口。当前操作按钮只显示阶段提示：产品编辑安排在2C3A，仓库移除安排在2C3B，真实库存操作安排在2C4。页面不会跳入旧的本地假库存操作。

详情页不展示mock流水，仅显示“真实库存流水将在后续阶段接入”。

## 创建返回刷新

库存页打开创建页时记录一次页面内存标记。创建成功通过 `wx.switchTab` 返回库存页后，`onShow` 消费该标记并重新读取真实第一页。创建响应不会被手工塞入列表，也不会写Storage；普通Tab切换不会无条件重复刷新。

## 上下文失效

团队、成员或仓库状态失效时，页面立即清除当前显示数据，调用现有 `clearTeamContext` 和强制bootstrap流程，然后返回startup重新路由。流程不使用mock角色，也不会无限重试。

## 云端变更

本阶段未修改 `warehouse-api`。正常情况下无需重新部署云函数，无需新增集合、索引或数据库权限；仍需保证阶段2C2A部署文档要求的组合索引已经生效。
