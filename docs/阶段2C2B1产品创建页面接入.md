# 阶段2C2B1产品创建页面接入

## 目标

阶段2C2B1将既有三步产品创建原型接入真实 `product.create`。此前“完成”按钮仅显示占位提示，没有调用 service，因此不会写入任何云端集合。

现在的调用链为：

`product-edit` 表单 → 本地字段转换与校验 → `productService.createProduct(payload)` → `cloudService.callApi('product.create')` → `warehouse-api` 云端事务。

页面不直接调用 `wx.cloud`，也不直接访问数据库。

## 字段映射

页面只提交以下白名单字段：

- `name` ← 产品名称
- `productCode` ← 产品编号
- `category` ← 分类
- `unit` ← 单位；选择“其他”时使用自定义单位
- `brand`、`specification`、`description` ← 去除首尾空格后的可选文本
- `minStock` ← 启用低库存提醒时的最低库存；关闭时为 `0`
- `initialStock` ← 原型表单中的初始库存 `stock`
- `coverType`、`coverText`、`coverEmoji`、`coverBackground` ← 封面设置
- `requestKey` ← 首次真实提交时生成的随机幂等键

页面不会提交团队、仓库、用户、角色、openId、服务端规范化字段、搜索关键词、库存状态、版本号或文档ID。身份和作用域由云函数重新确认，`searchKeywords` 由云函数生成。

数量字段只接受不超过 `999999999` 的非负安全整数；小数、负数、NaN和不安全整数在请求前被拒绝。

## 封面规则

- 文字封面映射为 `coverType: 'text'`，并提交文字和系统背景色。
- 系统图标映射为 `coverType: 'emoji'`，并提交受支持的emoji和系统背景色。
- 无封面映射为 `coverType: 'none'`。
- 本地图片仍可在原型中预览，但当前不会上传或提交临时路径。点击完成时会提示用户改用文字或表情封面，并保留表单内容。

## 幂等与保存状态

页面保存状态包含 `saving`、`saveError`、`createRequestKey`、`submittedPayloadHash` 和 `createdProduct`。

- 首次真实提交生成随机 `requestKey`。
- 保存期间忽略重复点击，并锁定完成和返回修改操作。
- 网络失败、超时或结果不确定时保留原请求键；相同表单重试会复用该键。
- 已发送后修改表单内容，下一次提交会生成新键。
- `REQUEST_KEY_CONFLICT` 会清除旧键，用户再次提交时生成新键。
- 成功后清除请求键，不写入 Storage，也不允许页面再次创建同一产品。
- 页面卸载后不会继续 `setData`。

## 云端写入结果

一次成功的事务会创建：

1. `products`：团队共享产品主数据。
2. `warehouse_products`：当前默认仓库中的产品关系、库存余额和最低库存。
3. `stock_records`：仅当 `initialStock > 0` 时创建一条 `initial` 流水。

当 `initialStock = 0` 时，`initialRecord` 合法地为 `null`，但前两个集合仍会新增文档。

## 权限与导航

owner和admin可以看到库存页新增入口并创建产品。viewer不显示新增入口；即使通过路由直接进入创建页，也会被页面拦截并返回库存页。云函数的 `FORBIDDEN` 校验仍是最终权限边界。

成功后显示“产品创建成功”，并通过 `wx.switchTab` 返回库存页。当前库存首页仍使用阶段1的mock列表，因此新产品不会立即显示；真实列表与详情接入安排在阶段2C2B2。

## 云端变更

本阶段没有修改 `warehouse-api`，正常情况下无需重新部署云函数，也无需新增集合、索引或数据库权限。
